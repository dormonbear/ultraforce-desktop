# SP-D sf-schema (thin slice) Implementation Plan

> Date: 2026-06-18 ¬∑ Crate: `crates/sf-schema` ¬∑ Depends on: sf-core
> Spec: specs/2026-06-18-sf-schema-design.md

Implements the thin schema slice: on-demand single-object describe ‚Üí trimmed
model ‚Üí disk + in-memory cache ‚Üí basic query. TDD throughout. Mirrors the SP-A
build style (MockRunner unit tests + a gated e2e).

## Global Constraints

- Dependency direction: `sf-schema ‚Üí sf-core` only. No other crate deps.
- Reuse `sf_core::{SfInvoker, SfError}` and `sf_core::runner::MockRunner`
  (behind sf-core's `test-util` feature) for tests ‚Äî do NOT spawn real `sf` in
  unit tests.
- `run_json` already appends `--json` and surfaces empty-stdout errors; call it,
  don't reach for `run_raw`.
- Trim the model to the keys in the spec; rely on serde ignoring unknown keys.
  `field_type` ‚Üê `#[serde(rename = "type")]`.
- A real trimmed describe fixture already exists at
  `crates/sf-schema/tests/fixtures/describe_account.json` (Account: Id, Name,
  Type[picklist], OwnerId[reference‚ÜíUser, rel "Owner"], Industry[picklist],
  + 2 child relationships). Use it as the canonical unit-test fixture.
- Every task is RED‚ÜíGREEN: write the failing test first, then the impl.
- Run `cargo test -p sf-schema` after each task; `cargo test --workspace` at the
  end. Do NOT run the `#[ignore]` e2e in the normal suite.

### Task 1: Scaffold the crate

- Add `crates/sf-schema` to the workspace `members`.
- `Cargo.toml`: `[dependencies]` sf-core (path), serde (derive), serde_json,
  tokio (rt-multi-thread, macros); `[dev-dependencies]` sf-core with
  `features = ["test-util"]`. **No new external crates** ‚Äî everything needed is
  already in the workspace lock (no `tempfile`, no `directories`).
- `src/lib.rs`: `pub mod model; pub mod puller; pub mod store; pub mod query;`
  and re-export the public types.
- Verify: `cargo build -p sf-schema`.

### Task 2: model.rs ‚Äî trimmed serde structs

Structs exactly per spec: `SObjectSchema`, `Field`, `PicklistValue`,
`ChildRelationship`. serde renames: `field_type`‚Üê`type`, `label_plural`‚Üê
`labelPlural`, `key_prefix`‚Üê`keyPrefix`, `reference_to`‚Üê`referenceTo`,
`relationship_name`‚Üê`relationshipName`, `picklist_values`‚Üê`picklistValues`,
`default_value`‚Üê`defaultValue`, `child_sobject`‚Üê`childSObject`,
`child_relationships`‚Üê`childRelationships`. Use `#[serde(default)]` on optional/
array fields so absent keys deserialize cleanly.

**Test (RED first):** deserialize `tests/fixtures/describe_account.json`'s
`result` object into `SObjectSchema`; assert `name == "Account"`, fields len == 5,
the `OwnerId` field has `reference_to == ["User"]` and `relationship_name ==
Some("Owner")`, the `Type` field has a non-empty `picklist_values`, and
`child_relationships` len == 2.

### Task 3: puller.rs ‚Äî describe_object

```rust
pub async fn describe_object(invoker: &SfInvoker, object: &str)
    -> Result<SObjectSchema, SfError> {
    invoker.run_json(&["sobject", "describe", "-s", object]).await
}
```

**Test:** `MockRunner` returning the fixture's full envelope string; assert
`describe_object(&invoker, "Account")` yields the same assertions as Task 2 and
that the args seen by the runner are `["sobject","describe","-s","Account","--json"]`.

### Task 4: query.rs ‚Äî lookups on SObjectSchema

`impl SObjectSchema`: `field(&self, name)` (case-insensitive match on
`field.name`), `picklist_values(&self, field)` (‚Üí `&[PicklistValue]`, empty if
field missing or non-picklist), `child_relationship(&self, name)`
(case-insensitive on `relationship_name`).

**Test:** on the fixture model ‚Äî `field("ownerid")` is Some; `field("nope")`
is None; `picklist_values("Type")` non-empty; `picklist_values("Name")` empty;
`child_relationship` resolves one of the two by name.

### Task 5: store.rs ‚Äî SchemaStore (disk + memory)

- `Key` = `(api_version, object_lowercased)`; mem = `HashMap<Key, SObjectSchema>`.
- `default_root()`: OS cache dir + `sf-toolkit`. Use `std::env`:
  `$XDG_CACHE_HOME` or `$HOME/.cache` on unix, `%LOCALAPPDATA%` on windows; fall
  back to `std::env::temp_dir()`. (No `directories` dep ‚Äî keep it lazy.)
- File path: `<cache_root>/<org_id>/<api_version>/<object>.json`. Sanitize
  `org_id`/`object` (replace path separators).
- `get` (mem only), `load_disk` (read+deserialize file ‚Üí mem, Ok(None) if
  absent), `get_or_fetch` (mem ‚Üí disk ‚Üí `describe_object` then persist pretty
  JSON + insert mem), `invalidate` (remove mem + `fs::remove_file`, ignore
  NotFound).

**Test (temp cache_root, MockRunner counting calls):** create the cache root
with `std::env::temp_dir().join(format!("sf-schema-test-{}-{}", std::process::id(),
nanos))` (unique per run), `fs::create_dir_all` it, and `fs::remove_dir_all` at the
end ‚Äî no `tempfile` crate.
- `get_or_fetch` describes once, second call hits mem (runner called once ‚Äî
  track with an `Arc<AtomicUsize>` in the MockRunner closure).
- A fresh `SchemaStore` over the same root: `load_disk("Account")` returns the
  persisted schema without any runner.
- `invalidate("Account")` deletes the file; next `get_or_fetch` re-describes
  (runner count increments).
