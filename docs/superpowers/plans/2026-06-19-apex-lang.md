# SP-F apex-lang (Phase 1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> Date: 2026-06-19 · Crate: `crates/apex-lang` · Depends on: sf-schema → sf-core
> Spec: specs/2026-06-19-apex-lang-design.md

In-process Apex completion via an Offline Symbol Table (OST). The OST is
GENERATED from the user's org through Salesforce first-party endpoints (that plugin
mechanism, our own data) — never per-keystroke `sf`. Phase 1 ships OST
acquisition + basic completion. TDD throughout.

## Goal

Build `crates/apex-lang` to: acquire an OST from three first-party sources
(Tooling `completions?type=apex`, sObject describe via `sf-schema`, `ApexClass`
SymbolTable), cache it on disk + in memory versioned by `(org, apiVersion)`,
lex Apex, classify cursor context, and complete top-level + `Type.`/`expr.`
member access against the cached OST. Inference and diagnostics are later phases.

## Architecture

`acquire` + `store` are the only sf-touching modules (via `SfInvoker`,
MockRunner-testable). `symbols` is the OST model. `lexer`, `parser`, `resolve`,
`complete` are PURE — borrowed OST + source string, zero IO. The `features`
layer (later) wires a real `SfInvoker` + cache root; `apex-lang` only reads the
OST it built.

## Tech Stack

Rust 2021, workspace member (auto-included via `crates/*` glob). serde/serde_json
for OST (de)serialization, thiserror via `sf_core::SfError`, tokio for async
acquisition. No new external crates.

### Task 2: symbols.rs — OST model (RED first)

- Structs (serde-derived, `#[serde(default)]` on arrays/optionals):
  - `Ost { namespaces: Vec<Namespace>, org_types: Vec<ApexType> }`
  - `Namespace { name: String, types: Vec<ApexType> }`
  - `ApexType { name: String, kind: TypeKind, methods: Vec<Method>,
    properties: Vec<Property>, enum_values: Vec<String> }`
  - `Method { name: String, return_type: String, params: Vec<String>,
    is_static: bool }`
  - `Property { name: String, prop_type: String, is_static: bool }`
  - `enum TypeKind { Class, Interface, Enum }` (serde `rename_all = "lowercase"`)
- Methods: `Ost::type_in(&self, namespace, name) -> Option<&ApexType>` and
  `Ost::org_type(&self, name) -> Option<&ApexType>` (both case-insensitive on
  type name); `ApexType::member(&self, name) -> Option<Member>` where `Member`
  borrows either a method or property by case-insensitive name.
- **Test:** construct an `Ost` in code with a `System` namespace containing a
  `String` class (method `valueOf`, static) and a `Database` class; assert
  `type_in("System","string")` is Some with `name == "String"`,
  `org_type("Foo")` is None, and `ApexType::member` finds `valueOf`
  case-insensitively. Also round-trip one `ApexType` through serde_json and
  assert an unknown extra key is ignored.
- Commit: `feat(apex-lang): OST symbol model`

### Task 4: store.rs — OstStore (disk + memory) (RED first)

- Mirror `sf_schema::SchemaStore`:
  - `enum OstSource { Stdlib, OrgTypes }` (file stem `"stdlib"` / `"org_types"`).
  - `OstStore { root: PathBuf, org_id: String, mem: HashMap<(String,OstSource),
    serde_json::Value> }`; `new(root, org_id)`; `default_root()` reusing the
    same OS-cache-dir logic as `SchemaStore::default_root` (no `directories`).
  - `file_path(api_version, source)` →
    `<root>/<org_id>/<api_version>/apex-ost/<stem>.json` (sanitize `org_id`).
  - `get` (mem), `load_disk` (read+deserialize → mem, `Ok(None)` if absent),
    `get_or_fetch(invoker, api_version, source)` (mem → disk → acquire+persist
    pretty JSON + insert mem), `invalidate(api_version, source)` (drop mem +
    `fs::remove_file`, ignore NotFound).
- **Test (temp root, counting MockRunner — same pattern as sf-schema store
  tests):** unique temp root per run via `std::env::temp_dir().join(format!(
  "apex-lang-test-{}-{}-{}", pid, nanos, seq))`, `create_dir_all`,
  `remove_dir_all` at end.
  - `get_or_fetch(Stdlib)` acquires once; second call hits mem (runner called
    once — `Arc<AtomicUsize>`).
  - Fresh `OstStore` over the same root: `load_disk(api, Stdlib)` returns the
    persisted value without a runner call.
  - `invalidate(api, Stdlib)` deletes the file; next `get_or_fetch` re-acquires
    (count increments). `OrgTypes` invalidation does NOT evict `Stdlib`.
- Commit: `feat(apex-lang): versioned OST disk+memory cache`

### Task 5: lexer.rs — Apex lexer with spans (RED first)

- `enum TokenKind { Keyword, Ident, Dot, LParen, RParen, LBrace, RBrace,
  Semicolon, Comma, Lt, Gt, Literal, Whitespace, Other }`;
  `Token { kind, text, start, end }`; `pub fn lex(&str) -> Vec<Token>`.
- Tokenize: identifiers `[A-Za-z_][A-Za-z0-9_]*` (classify `Keyword` against a
  case-insensitive Apex keyword set — `new`, `return`, `if`, `for`, `void`,
  `public`, `private`, `static`, etc.; else `Ident`); `.`→Dot, `(`→LParen,
  `)`→RParen, `{`→LBrace, `}`→RBrace, `;`→Semicolon, `,`→Comma, `<`→Lt, `>`→Gt;
  `'…'` string and numeric runs → `Literal`; whitespace runs → Whitespace;
  anything else → Other. Track byte spans.
- **Test:** `lex("Integer x = String.valueOf(1);")` → kinds in order
  Ident(Integer), Ws, Ident(x), Ws, Other(=), Ws, Ident(String), Dot,
  Ident(valueOf), LParen, Literal(1), RParen, Semicolon; assert the span of
  `"String"` slices back to `"String"`. `lex("new ")` → Keyword(new).
- Commit: `feat(apex-lang): Apex lexer with spans`

### Task 6: parser.rs — outline + cursor context (RED first)

- `LocalVar { name: String, declared_type: String }`;
  `ApexOutline { locals: Vec<LocalVar> }`; `pub fn outline(&str) -> ApexOutline`:
  walk tokens, recognize the simple `Ident Ident (= …)? ;` declaration pattern
  (first Ident = type, second = name) → collect `locals`. Ignore everything else
  this slice.
- `enum CursorContext { TopLevel { prefix: String }, StaticMember { type_name:
  String, prefix: String }, InstanceMember { receiver: String, prefix: String },
  Unknown }`; `pub fn context_at(&str, cursor: usize) -> CursorContext`:
  walk back from `cursor` over `[A-Za-z0-9_]` to get `prefix`; if the token
  immediately before the prefix is a `Dot`, the token before that Dot is the
  receiver → if it is a known-type-shaped Ident emit `StaticMember`, else
  `InstanceMember`; otherwise `TopLevel`.
- **Test:** `outline("Account a; Integer n = 0;")` → locals
  `[(Account,a),(Integer,n)]`. `context_at("String.valO", after "valO")` →
  `StaticMember{ type_name:"String", prefix:"valO" }`. `context_at("a.nam",
  after "nam")` → `InstanceMember{ receiver:"a", prefix:"nam" }`.
  `context_at("Inte", after "Inte")` → `TopLevel{ prefix:"Inte" }`.
- Commit: `feat(apex-lang): outline locals + cursor context classification`

### Task 7: resolve.rs — name → OST symbol (basic) (RED first)

- `pub fn resolve_type<'a>(ost:&'a Ost, name:&str) -> Option<&'a ApexType>`:
  search `org_types` first, then every namespace's `types`, case-insensitive.
- `pub fn resolve_receiver_type<'a>(ost:&'a Ost, outline:&ApexOutline,
  receiver:&str) -> Option<&'a ApexType>`: if `receiver` matches a local's name,
  resolve that local's `declared_type` via `resolve_type`; else treat
  `receiver` as a type name directly. (No expression inference — Phase 2.)
- **Test (in-code OST + outline):** `resolve_type(ost,"string")` → the `String`
  type. With outline `[(Account,a)]`, `resolve_receiver_type(ost,outline,"a")`
  → the `Account` type; `resolve_receiver_type(ost,outline,"String")` → the
  `String` type; unknown receiver → None.
- Commit: `feat(apex-lang): basic type + receiver resolution`

### Task 8: complete.rs — completion against the OST (RED first)

- `enum CandidateKind { Type, Keyword, LocalVar, Method, Property }`;
  `Candidate { label: String, kind: CandidateKind }`;
  `pub fn complete(input:&str, cursor:usize, ost:&Ost) -> Vec<Candidate>`:
  build `outline` + `context_at`, then:
  - `TopLevel{prefix}` → prefix-filtered (case-insensitive) candidates from:
    all OST type names (`Type`), a small keyword set (`Keyword`), and outline
    locals (`LocalVar`). Sorted by label, deduped.
  - `StaticMember{type_name,prefix}` → resolve the type; emit its `is_static`
    methods (`Method`) + `is_static` properties (`Property`) whose name matches
    the prefix.
  - `InstanceMember{receiver,prefix}` → `resolve_receiver_type`; emit its
    non-static members matching the prefix.
  - `Unknown` → `[]`.
- **Test (in-code OST: System.String{static valueOf}, System.Database; org type
  AccountService{instance method save}; outline local `AccountService svc`):**
  - `complete("Stri", 4, ost)` → labels contain `"String"` (kind Type), not
    `"Database"`.
  - `complete("String.val", 10, ost)` → contains `"valueOf"` (kind Method).
  - `complete("AccountService svc; svc.sa", cursor_after "sa", ost)` →
    contains `"save"` (instance Method).
  - cursor in dead space → `[]`.
- Commit: `feat(apex-lang): top-level + member-access completion`

## Phase 2 (outline)

> To be detailed into full TDD tasks AFTER Phase 1 lands and the OST shape is
> pinned. Tasks named with one-line intent only.

- **infer-local-types** — infer a local's type from its initializer expression
  (`new X()`, literals, method-return types) instead of only the declared type.
- **infer-expression-result** — resolve the result type of a member-access /
  method-call chain (`a.getContact().Name`) for `expr.` completion.
- **overload-resolution** — pick the right `Method` among overloads by arity /
  argument types for return-type inference.
- **generics-collections** — model and resolve element types of `List<T>`,
  `Set<T>`, `Map<K,V>` so `myList.` and `myMap.get(k).` complete correctly.
- **resolve-uses-inference** — rewire `resolve_receiver_type` to consume the
  inference engine, replacing Phase 1's declared-type-only path.

## Phase 3 (outline)

> To be detailed into full TDD tasks AFTER Phase 2. Tasks named with one-line
> intent only. Desktop wiring will likely become its own desktop slice.

- **diagnostics-unknown-symbol** — flag identifiers that resolve to no OST
  symbol (pure, mirrors `soql-lang::diagnostics`).
- **diagnostics-unknown-type** — flag declared/referenced types absent from the
  OST.
- **diagnostics-unknown-method** — flag member-access on a resolved type whose
  member does not exist.
- **soql-in-apex** — detect inline `[SELECT …]` blocks and delegate them to
  `soql-lang` for completion/diagnostics.
- **desktop-tauri-command** — a Tauri command exposing OST completion (likely a
  new desktop slice, not in `apex-lang`).
- **desktop-monaco-provider** — a Monaco `CompletionItemProvider` calling the
  Tauri command (same desktop slice).
