# SP-E soql-lang (thin slice) Implementation Plan

> Date: 2026-06-18 · Crate: `crates/soql-lang` · Depends on: sf-schema
> Spec: specs/2026-06-18-soql-lang-design.md

In-process SOQL completion + diagnostics. Pure (no IO). TDD throughout.

## Global Constraints

- New crate `crates/soql-lang`; add to workspace `members`.
- `[dependencies]`: `sf-schema = { path = "../sf-schema" }`, `serde` only if
  needed (probably not). NO sf-core in normal deps, NO IO. `[dev-dependencies]`:
  `sf-core = { path = "../sf-core" }`, `sf-schema = { path = "../sf-schema" }`
  (for the e2e), `tokio = { workspace = true }`.
- `complete`/`diagnostics` are PURE: they take `&sf_schema::SObjectSchema`. No
  network, no registry, no `sf`.
- Every task RED→GREEN; `cargo test -p soql-lang` after each; `cargo test
  --workspace` at the end; clippy `-D warnings` + `cargo fmt -p soql-lang --check`.
- Do NOT run the `#[ignore]` e2e in the normal suite. No new external crates.

Pre-verified API facts:
- `sf_schema::SObjectSchema` has public `name: String`, `fields: Vec<Field>`,
  and method `field(&self, name: &str) -> Option<&Field>` (case-insensitive).
  `sf_schema::Field` has public `name: String`. `sf_schema::describe_object(
  &SfInvoker, &str) -> Result<SObjectSchema, SfError>` for the e2e.

### Task 1: Scaffold crate
- `crates/soql-lang/{Cargo.toml, src/lib.rs}`; modules `lexer parse complete
  diagnostics`; add to workspace members.
- Verify `cargo build -p soql-lang`.
- Commit: `chore(soql-lang): scaffold crate`

### Task 2: lexer
- `TokenKind`, `Token{kind,text,start,end}`; `pub fn lex(&str)->Vec<Token>`.
- Tokenize: identifiers `[A-Za-z_][A-Za-z0-9_]*` (classify as `Keyword` if it
  matches the case-insensitive keyword set in the spec, else `Ident`), `,`→Comma,
  `.`→Dot, `(`→LParen, `)`→RParen, `*`→Star, runs of whitespace→Whitespace,
  anything else→Other. Track byte spans (`start`/`end`).
- **Test:** `lex("SELECT Id, Name FROM Account")` → kinds in order
  Keyword(SELECT), Ws, Ident(Id), Comma, Ws, Ident(Name), Ws, Keyword(FROM), Ws,
  Ident(Account); assert the span of "Account" slices back to "Account".
- Commit: `feat(soql-lang): SOQL lexer with spans`

### Task 3: outline (parse-lite)
- `SoqlOutline{from_object:Option<String>, select_fields:Vec<FieldRef>}`,
  `FieldRef{name,start,end}`. `pub fn outline(&str)->SoqlOutline`.
- Walk non-whitespace tokens: after a `FROM` keyword, the next `Ident` is
  `from_object`. Between `SELECT` and `FROM`, collect `Ident` (joining
  `Ident Dot Ident…` runs into a single dotted `name` with a span covering the
  run; a lone `*` Star is ignored for select_fields). Stop the select list at
  `FROM`.
- **Test:** `outline("SELECT Id, Name FROM Account")` → from_object Some("Account"),
  select_fields names ["Id","Name"] with spans that slice back correctly;
  `outline("SELECT Owner.Name FROM Account")` → one field "Owner.Name";
  `outline("SELECT Id")` → from_object None.
- Commit: `feat(soql-lang): outline FROM object and SELECT field list`

### Task 4: clause_at + complete
- `Clause{Select,From,Other}`; `pub fn clause_at(&SoqlOutline,&str,cursor)->Clause`
  (cursor byte offset → which region: between SELECT and FROM = Select; at/after
  the FROM object slot = From; else Other).
- `Candidate{label,kind}`, `CandidateKind::Field`. `pub fn complete(input,cursor,
  &SObjectSchema)->Vec<Candidate>`: if `clause_at` is Select, take the partial
  identifier ending at `cursor` (walk back over `[A-Za-z0-9_]`), return
  `schema.fields` whose `name` starts with it case-insensitively as `Field`
  candidates, sorted by name, deduped. Else `[]`.
- **Test (in-code schema** name "Account", fields Id, Name, Industry, OwnerId**):**
  for input `"SELECT Na FROM Account"` with cursor right after "Na" → labels
  contain "Name", not "Id". Cursor inside "Account" → `[]`. Cursor right after
  "SELECT " (empty partial) → all four field names.
- Commit: `feat(soql-lang): SELECT field-name completion`

### Task 5: diagnostics
- `Diagnostic{message,start,end,severity}`, `Severity{Error,Warning}`.
  `pub fn diagnostics(input,&SObjectSchema)->Vec<Diagnostic>`: build outline; if
  `from_object` is None → `[]`. For each select field whose `name` has no `.`
  and is not `*` and `schema.field(name).is_none()` → Error
  `format!("Unknown field '{name}' on {}", schema.name)` at its span.
- **Test:** `diagnostics("SELECT Id, Bogus FROM Account", &schema)` → one Error
  whose span slices to "Bogus"; `"SELECT Id, Name FROM Account"` → empty;
  `"SELECT Owner.Name FROM Account"` → empty (dotted skipped).
- Commit: `feat(soql-lang): unknown-field diagnostics`

### Task 6: gated e2e
- `tests/e2e.rs`, `#[ignore]`, `#[tokio::test]`: `SfInvoker::new(Arc::new(
  ProcessRunner))`, `sf_schema::describe_object(&invoker,"Account")`, then
  `complete("SELECT Nam FROM Account", cursor_after_Nam, &schema)` asserts a
  "Name" candidate, and `diagnostics("SELECT NotARealField123 FROM Account",
  &schema)` asserts exactly one Error.
- Commit: `test(soql-lang): gated e2e completion against real Account schema`
